# GitHub Releases Page Guide

[Русская версия](#ru)

This guide describes how to keep the GitHub Releases page clean and useful for GuitarToolkit users.

For many users, the Releases page is the real download page. It should be simple, clear, and trustworthy.

## Goals

A good GuitarToolkit release page should answer:

- what changed;
- what to download;
- how to install Desktop;
- how to install VST3;
- what is known to be limited or unstable;
- where to ask for help.

## Required assets

Every public release should include:

```text
GuitarToolkit_DESKTOP_v.<version>.zip
GuitarToolkit_VST3_v.<version>.zip
```

The release should not rely only on source code archives automatically generated by GitHub.

## Asset naming

Use consistent names:

```text
GuitarToolkit_DESKTOP_v.1.6.0.zip
GuitarToolkit_VST3_v.1.6.0.zip
```

Avoid:

```text
release.zip
plugin.zip
build.zip
new.zip
final.zip
final2.zip
```

## Release title

Recommended format:

```text
GuitarToolkit v1.6.0
```

For pre-releases:

```text
GuitarToolkit v1.6.0-beta.1
```

## Release description structure

Recommended structure:

```md
# GuitarToolkit v<version>

## Highlights

- ...
- ...
- ...

## Downloads

- `GuitarToolkit_DESKTOP_v.<version>.zip` - standalone Windows desktop app.
- `GuitarToolkit_VST3_v.<version>.zip` - VST3 plugin package for DAW hosts.

## Installation notes

For Desktop, extract the ZIP and run the desktop executable.

For VST3, copy the whole `GuitarToolkit` plugin folder to:

`C:\Program Files\Common Files\VST3\GuitarToolkit\`

Do not copy only the `.vst3` file. The plugin needs its DLL dependencies and the `runtimes` folder.

## Verification

- Build: passed.
- Tests: passed.
- Desktop smoke test: passed / not checked.
- VST3 smoke test: passed / not checked.

## Known issues

- ...

## Links

- Quick Start: https://github.com/LuTiK1984/GuitarToolkit/blob/master/docs/QUICK_START.md
- Discussions: https://github.com/LuTiK1984/GuitarToolkit/discussions
- Supported DAWs: https://github.com/LuTiK1984/GuitarToolkit/blob/master/docs/SUPPORTED_DAWS.md
```

## Latest release checklist

The latest release should have:

- clear title;
- short useful description;
- Desktop ZIP;
- VST3 ZIP;
- install notes;
- known issues;
- verification notes;
- links to Quick Start and Discussions.

## What to avoid

Avoid release pages that contain only:

- auto-generated GitHub source archives;
- no install instructions;
- no VST3 folder warning;
- vague text such as `fixes` or `update`;
- outdated repository links;
- missing Desktop or VST3 ZIP.

## Suggested description for a repository-polish release

Use this for a release focused on GitHub/project polish rather than app features:

```md
# GuitarToolkit v1.6.0

This release focuses on project polish, documentation, release hygiene, CI improvements, and open-source community setup.

The main app features are not the focus of this release, but the repository is now easier to use, maintain, and contribute to.

## Highlights

- Added GitHub Discussions setup and bilingual discussion forms.
- Added Support, Quick Start, Dependency Policy, Release Process, Project Structure, Maintainer Guide, GitHub Settings, and Branch Protection documentation.
- Improved README navigation and repository links after the project rename.
- Cleaned duplicate setup/install/release-note files from the repository root.
- Expanded `.gitignore` to prevent committing build outputs, logs, test results, release archives, and local files.
- Improved CI with a manual release package check workflow.
- Updated release packaging to include current documentation files.

## Downloads

- `GuitarToolkit_DESKTOP_v1.6.0.zip` - standalone Windows desktop app.
- `GuitarToolkit_VST3_v1.6.0.zip` - VST3 plugin package for DAW hosts.

## Installation notes

For Desktop, extract the ZIP and run the desktop executable.

For VST3, copy the whole `GuitarToolkit` plugin folder to:

`C:\Program Files\Common Files\VST3\GuitarToolkit\`

Do not copy only the `.vst3` file.

## Links

- Quick Start: https://github.com/LuTiK1984/GuitarToolkit/blob/master/docs/QUICK_START.md
- Discussions: https://github.com/LuTiK1984/GuitarToolkit/discussions
- Supported DAWs: https://github.com/LuTiK1984/GuitarToolkit/blob/master/docs/SUPPORTED_DAWS.md
```

## Cleanup routine for old releases

Do not delete old releases without a reason. Old releases can be useful for users who need to compare behavior.

Instead:

- make sure old releases have correct assets;
- mark broken releases as pre-release or add a warning if needed;
- keep the latest release polished;
- avoid uploading duplicate or confusing assets;
- remove accidentally uploaded local artifacts if they are not useful.

## Manual GitHub steps

To edit a release:

```text
Repository -> Releases -> Choose release -> Edit
```

To upload assets:

```text
Repository -> Releases -> Choose release -> Edit -> Attach binaries
```

To mark as latest:

```text
Repository -> Releases -> Choose release -> Set as latest release
```

# Руководство по странице GitHub Releases

[English version](#github-releases-page-guide)

Этот документ описывает, как держать страницу GitHub Releases чистой и полезной для пользователей GuitarToolkit.

Для многих пользователей Releases — это фактически страница загрузки. Она должна быть простой, понятной и вызывающей доверие.

## Цели

Хорошая release page должна отвечать:

- что изменилось;
- что скачивать;
- как установить Desktop;
- как установить VST3;
- какие есть известные ограничения;
- где просить помощь.

## Обязательные assets

Каждый публичный релиз должен включать:

```text
GuitarToolkit_DESKTOP_v.<version>.zip
GuitarToolkit_VST3_v.<version>.zip
```

Релиз не должен полагаться только на source code archives, которые GitHub создаёт автоматически.

## Имена assets


## Чеклист latest release

Последний релиз должен иметь:

- понятное название;
- короткое полезное описание;
- Desktop ZIP;
- VST3 ZIP;
- install notes;
- known issues;
- verification notes;
- ссылки на Quick Start и Discussions.

## Чего избегать

Плохо, если release page содержит только:

- auto-generated GitHub source archives;
- нет install instructions;
- нет предупреждения про полную VST3 folder;
- vague text вроде `fixes` или `update`;
- устаревшие ссылки на repository;
- нет Desktop или VST3 ZIP.

Не удаляйте старые релизы без причины. Они могут быть полезны пользователям для сравнения поведения.
