# Support

[Русская версия](#ru)

Use this page to choose the right place for your question, report, or idea.

## Where to ask

| Need | Use |
| --- | --- |
| Installation help, setup questions, build questions, or general usage help | GitHub Discussions -> Q&A |
| VST3 behavior in FL Studio, Reaper, Cubase, Ableton Live, or another DAW | GitHub Discussions -> DAW Compatibility |
| Feature ideas, workflow improvements, or module suggestions | GitHub Discussions -> Ideas |
| Screenshots, demos, setups, or examples of using GuitarToolkit | GitHub Discussions -> Show and Tell |
| Reproducible bugs with clear steps | GitHub Issues -> Bug report |
| Security-sensitive reports | Follow `SECURITY.md` |

## Before opening a bug report

Please check:

- the latest release is installed;
- the full VST3 plugin folder was copied, not only the `.vst3` file;
- the DAW plugin scan was refreshed after installation;
- the issue can be reproduced more than once;
- logs from `%AppData%\GuitarToolkit\logs` are available if the app or plugin failed.

## Useful details for VST3 support

When asking about VST3 or DAW behavior, include:

- GuitarToolkit version;
- DAW name and version;
- Windows version;
- audio interface or input device;
- sample rate and buffer size, if relevant;
- whether the plugin was freshly scanned;
- steps you tested;
- logs or screenshots, if available.

## Scope

GuitarToolkit is a small open-source project. Support is best-effort and focused on clear, reproducible reports and practical musician workflows.

## Перед созданием bug report

Проверьте:

- установлена последняя версия релиза;
- скопирована вся папка VST3-плагина, а не только `.vst3` файл;
- после установки DAW пересканировала плагины;
- проблема воспроизводится больше одного раза;
- если приложение или плагин упали, доступны логи из `%AppData%\GuitarToolkit\logs`.

## Полезные детали для VST3-поддержки

Если вопрос связан с VST3 или DAW, укажите:

- версию GuitarToolkit;
- название и версию DAW;
- версию Windows;
- аудиоинтерфейс или входное устройство;
- sample rate и buffer size, если важно;
- был ли плагин заново просканирован;
- проверенные шаги;
- логи или скриншоты, если есть.

## Границы поддержки

GuitarToolkit — небольшой open-source проект. Поддержка оказывается по возможности и ориентирована на понятные воспроизводимые отчёты и практичные сценарии для музыкантов.
